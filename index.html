Projects

Virtual Walking Tour - University of Cincinnati



Created life-like, interactive, responsive and scalable web based Virtual Tour

Created responsive web application to provide useful information about university to upcoming students. It contained photo gallery, video gallery and virtual tour for public access.

In virtual tour, one can walk through places in campus using forward, backward, left and right arrows present on pathways of each image as shown in picture. User can also see text or listen to audio associated with each view. Google Maps API was also integrated to provide more information for each place.

All the images for different directions in tour were clicked manually and saved in Sql Server database in byte code. Graph algorithm was used to map images based on directions and store in database. Based on type of arrow clicked by user, images were retrieved from db using wcf service. It was then converted to base64string in C# controller and passed to front-end for display.

Additionally, admin portal was created to add more buildings and its related pictures. If new building is added to the university, Admin can upload the images direction wise. Newly created audio can also be added to the list dynamically.

This entire project was developed by me and a class-mate in one week for the course, XML and Web Services and received ‘A’ grade in it.

Technologies Used: C#, MVC, WCF, XML, HTML, CSS, Bootstrap, jQuery, JavaScript, Ajax, Entity Framework, SQL, SQL Server, .Net, Visual Studio

Cure @Home for pregnant women

  • In India, 78,000 mothers die every year due to lack of availability of instant primary medical advice and treatment. So in order to avoid such mishappenings, I developed an online portal to provide instant medical advice to pregnant women
  • Through the use of “latent Dirichlet allocation (LDA)” and “LexRank algorithm” model along with web crawling and parsing algorithms, a system was designed to allocate most suitable medical practitioner to a patient based on the information provided by a mother in her profile.
  • A patient can login into the system, check suggested doctor’s profile or select a suitable doctor by filling out various parameters in the provided search engine. She can also raise queries and seek advice from other online available doctors through chat system and by sharing images.
  • A safe payment system was developed using Advanced Encryption Standard (AES) algorithm to support online transactions in case a fee is prescribed by a doctor.
  • A patient can also maintain a medical history for future references

Disease Prediction Model

  • In developing countries like India, population residing in rural areas usually suffers from non-availability of basic and quality healthcare facilities leading to large number of deaths each year due to diseases like Tuberculosis, Malaria, Asthma etc. In order to overcome this problem, the data for past 5 years was collected to estimate the number of patients, who can suffer from the same disease in the upcoming 6th year through RBF (Radial Basis Function) algorithm so that health management organizations as well as government can be prepared in advance and provide required medical facilities on time.
  • The number of patients is extracted by many factors, for example, name, age, location, medical prescription, diagnosis, hospital address, medications, lab results, cost of treatment etc. from large volumes of database through RBF model based on Gaussian function.
